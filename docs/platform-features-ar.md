# ميزات المنصة وشرح المشروع

**الغرض:** تلخيص **ما يفعله المشروع** للمتسوق وللفريق التقني، و**القدرات المضافة** (CMS، لوحة تحكم، إشعارات ويب) مقابل أفكار مستقبلية مذكورة في [رؤية المشروع](project-vision.md).

---

## 1. ما هو المشروع؟

**سوكانى ستور (Sokany Store)** واجهة متجر **Headless**: الكتالوج والطلبات والدفع مرتبطة بـ **WooCommerce** عبر REST، بينما الواجهة مبنية بـ **Next.js (App Router)** وتُنشر كتطبيق ويب سريع و**جاهز للتثبيت كـ PWA** (عربي RTL، موبايل أولاً).

| الطبقة | الدور |
|--------|--------|
| **المتصفح / العميل** | صفحات المتجر، السلة، الدفع، الحساب — بدون مفاتيح WooCommerce السرية. |
| **Next.js API (`/app/api/*`)** | طبقة BFF: تتحدث مع WooCommerce من الخادم فقط، وتعرّف عقود بيانات مع **Zod**. |
| **WordPress + WooCommerce** | مصدر الحقيقة للمنتجات، الفئات، الطلبات، والدفع. |

التفاصيل التقنية للأداء والتخطيط (CLS، الكروم التجاري السفلي، البحث) مذكورة في [`tech-audit.md`](tech-audit.md).

---

## 2. ميزات واجهة المتجر (للمتسوق)

| النطاق | الوصف |
|--------|--------|
| **الرئيسية** | هيرو، عروض، أقسام، بانرات، مزج محتوى من **CMS** عند التوفر. |
| **الكتالوج** | `/products`، التصنيفات، تصفية وترتيب، بحث سياقي محسّن للتنقل. |
| **المنتج** | معرض صور، مراجعات (مع تحديثات تفاؤلية)، إضافة للسلة. |
| **السلة والدفع** | سلة مع حفظ محلي آمن للتجميع، إتمام الطلب نحو WooCommerce. |
| **الحساب** | تسجيل/دخول؛ ربط مع **Firebase** للتحقق من الهاتف (حسب الإعداد). |
| **طلباتي والتتبع** | عرض الطلبات ومتابعة الحالة. |
| **الأمنيات، من نحن، الفروع، الموزّعون** | صفحات ثقة ومحتوى؛ بيانات الفروع/الموزّعين قابلة للتحرير من **لوحة التحكم** عند تفعيل CMS. |
| **PWA** | `manifest` للتثبيت والأيقونات والسم؛ صفحة **offline** عبر service worker مُولَّد. |

## 3. المحتوى الديناميكي (CMS) — Firestore

بدل تعديل الكود لكل تغيير في الشعار أو الشرائح أو الفروع، يُخزَّن جزء من «محتوى الواجهة» في **Firestore** (مجموعة مخصصة للمتجر)، ويُقرأ على الخادم مع **تخزين مؤقت ووسم إبطال** عند الحفظ من لوحة التحكم.

**أمثلة على ما يمكن ضبطه من CMS (عند التفعيل):**

- إعدادات عامة: شريط إعلان علوي، عروض محدودة الوقت (عدّ تنازلي)، كلمات بحث سريعة.
- **هوية الموقع:** شعار، ألوان/وصف PWA، صور OG، إلخ (مدموجة مع متغيرات البيئة).
- شرائح **الهيرو**، **بانرات الأقسام**، **الفروع** (مبيعات/خدمة)، **الموزّعون**، **إعلانات مميزة (Spotlights)**.
- **الوسائط:** رفع صور إلى **Firebase Storage** لاستخدامها في الحقول النصية للروابط.

إذا لم يكن Firebase مهيأ أو لا توجد وثائق، يُستخدم **محتوى احتياطي** (ثوابت/بيانات مدمجة) دون كسر الصفحة.

## 4. لوحة التحكم (`/control`)

مسار **محمي** لمسؤولي المحتوى: تسجيل دخول منفصل، جلسة خادم، ثم تحميل وحفظ حزمة **CMS** عبر `/api/control/cms`.

| التبويب (أمثلة) | الوظيفة |
|------------------|---------|
| **عام** | الإعلان العلوي، العروض، الروابط الاجتماعية، كلمات البحث السريعة. |
| **هوية الموقع** | الشعار والظهور في metadata و PWA. |
| **الهيرو / البانرات / الفروع / الموزّعون / الإعلانات المميزة** | نماذج بسيطة مع تحقق **Zod** قبل الحفظ. |
| **الوسائط** | رفع صورة ونسخ الرابط. |
| **إشعارات** | إرسال **إشعار Web Push** عبر **FCM** إلى موضوع (مثل `all_customers`) للمشتركين الذين وافقوا — من الخادم باستخدام **Firebase Admin**. |

**متطلبات تشغيل لوحة التحكم:** بيانات اعتماد Firebase Admin (`FIREBASE_SERVICE_ACCOUNT_JSON`) وتهيئة المشروع في Firebase؛ راجع أيضًا [`firebase-web-push-vapid.md`](firebase-web-push-vapid.md) لمفتاح **VAPID** على الويب.

## 5. PWA وإشعارات الويب (FCM)

- **Service worker** يُولَّد من [`app/api/pwa-sw/route.ts`](../app/api/pwa-sw/route.ts): تخزين صفحة offline، أيقونة افتراضية، ودمج **Firebase Messaging** لاستقبال الإشعارات في الخلفية وعرضها كنظام إشعارات للمتصفح.
- **العميل:** الـ hook `useFcmWebPush` يطلب الإذن، يسجّل **رمز FCM**، ويرسله إلى `/api/push/subscribe`؛ الرسائل في المقدمة يمكن أن تظهر كـ toasts.
- **الخادم:** إرسال الإشعارات من لوحة التحكم أو من أي منطق يستخدم Admin SDK مع **Web Push** وموضوعات FCM.

**ملاحظة:** مفتاح VAPID يجب أن يكون من **Web Push certificates** في Firebase Console، وليس مفتاح API عام.

## 6. الفرق بين «رؤية المنتج» و«ما هو منفّذ اليوم»

في [`project-vision.md`](project-vision.md) يوجد جدول **Roadmap** (بحث سياقي أعمق، upselling ذكي، تكامل تشغيلي مع الخرائط، إلخ) — هذه **أفكار اتجاه** وليست بالضرورة كلها مُنفَّذة بالكامل في الكود.

**أمثلة على قدرات **منفّذة** في المستودع الحالي:**

- Headless WooCommerce + Zod + TanStack Query + Zustand.
- CMS عبر Firestore + لوحة تحكم + رفع وسائط.
- PWA مع offline + FCM للويب (مع إعداد صحيح).
- تحسينات UX موثقة في `tech-audit.md` (بحث، قائمة موبايل، كروم سفلي).
