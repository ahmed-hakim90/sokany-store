# صور بانر «حصرياً» لكل قسم أب (Home)

الخادم يقرأ المجلد عبر
[`features/home/services/getBannerSectionOrderedImages.ts`](../../../features/home/services/getBannerSectionOrderedImages.ts)
ويرجع **قائمة مرتبة** من المسارات؛ الصفحة تربط **أول قسم أب** بأول صورة، الثاني بالثانية، وهكذا.

## التسمية والترتيب

- ابدأ اسم الملف برقم بالترتيب المرغوب (يفضّل بادئة مثل `01`، `02`، `03`):

```
public/images/banner-section/
  01.webp
  02-kitchen.jpg
  03-personal-care.png
```

- الترتيب يعتمد على **الرقم في بداية الاسم** (بعدها يمكن وصف اختياري بعد `-`).
- لو وُجد `01.jpg` و`01.webp` معاً لنفس الرقم، يُختار الامتداد الأفضل:  
  `webp` → `avif` → `jpg` → `jpeg` → `png` → `gif`

## الملفات بدون رقم في البداية

تُرتّب أبجدياً **بعد** كل الملفات المرقّمة (للاستخدام النادر فقط).

## السلوك عند غياب صورة لترتيب معيّن

إن كان عدد الأقسام الأب أكبر من عدد الصور، الأقسام الزائدة تستخدم كما كان: صورة التصنيف من الـ API أو أول منتج ثم placeholder.

## ملاحظات

- الملفات التي تبدأ بنقطة (`.`) تُتجاهل.
- `README.md` لا يُعتبر صورة.
